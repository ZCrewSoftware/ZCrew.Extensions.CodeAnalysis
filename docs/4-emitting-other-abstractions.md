# Emitting Other Abstractions

The `[Microsoft.CodeAnalysis.Embedded]` attribute is not limited to attribute types. You can mark **any** type kind -- enum, class, struct, record, or interface -- and the library will generate a `SourceText` class for it. This lets your source generator embed shared types into consuming projects.

## Example: Embedding an Enum

Define an enum in your source generator project:

```csharp
namespace MyGenerator;

[Microsoft.CodeAnalysis.Embedded]
internal enum ServiceLifetime
{
    Transient,
    Scoped,
    Singleton,
}
```

The library generates a single file: `ServiceLifetimeSourceText`.

### Generated: `ServiceLifetimeSourceText`

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

using System.Text;
using Microsoft.CodeAnalysis.Text;

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal static class ServiceLifetimeSourceText
{
    public static readonly SourceText SourceText = SourceText.From(
        """
        namespace MyGenerator;

        [Microsoft.CodeAnalysis.Embedded]
        internal enum ServiceLifetime
        {
            Transient,
            Scoped,
            Singleton,
        }

        """,
        Encoding.UTF8
    );
}
```

The generated class exposes the type's full source code as a `SourceText` instance. The naming follows the pattern `{TypeName}SourceText`.

> Non-attribute types only generate the `SourceText` file. The attribute parsing infrastructure (`IDataBuilder`, `Constructor`, `Parameter`, etc.) is only generated for types that inherit from `System.Attribute`.

## Using the SourceText

### In PostInitializationOutput

Use `RegisterPostInitializationOutput` when the type should always be emitted, regardless of user code:

```csharp
public void Initialize(IncrementalGeneratorInitializationContext context)
{
    context.RegisterPostInitializationOutput(static context =>
    {
        context.AddSource("ServiceLifetime.g.cs", ServiceLifetimeSourceText.SourceText);
    });
}
```

### In RegisterSourceOutput

Use `RegisterSourceOutput` when the type should only be emitted conditionally, based on pipeline data:

```csharp
context.RegisterSourceOutput(someProvider, static (context, data) =>
{
    if (data.NeedsLifetimeEnum)
    {
        context.AddSource("ServiceLifetime.g.cs", ServiceLifetimeSourceText.SourceText);
    }
});
```

## Other Type Kinds

The same pattern works for any type kind:

```csharp
// A marker interface
[Microsoft.CodeAnalysis.Embedded]
internal interface IGeneratedService { }

// A base class
[Microsoft.CodeAnalysis.Embedded]
internal abstract class GeneratedServiceBase
{
    public abstract string Name { get; }
}

// A record struct
[Microsoft.CodeAnalysis.Embedded]
internal readonly record struct ServiceDescriptor(string Name, Type ServiceType);
```

Each produces a corresponding `{TypeName}SourceText` class with the relevant `SourceText` property.

## Best Practices

- **Keep embedded types self-contained.** Embedded types are emitted as source text into consuming projects. They should not depend on types that won't be available in those projects.
- **Avoid dependencies on other embedded types.** If type A references type B, both must be embedded and both must be emitted. Consider whether the coupling is necessary.
- **Use `internal` visibility.** Embedded types are implementation details of your source generator. Keeping them `internal` prevents them from leaking into the consuming project's public API.
