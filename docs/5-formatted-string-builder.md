# FormattedStringBuilder

`FormattedStringBuilder` is a `StringBuilder` wrapper that automatically manages indentation. Every call to `AppendLine()` appends the line terminator followed by the current indentation whitespace, so subsequent content starts at the correct indent level.

**Namespace:** `ZCrew.Extensions.CodeAnalysis.CSharp.Text`

## Core API

### Indentation

```csharp
FormattedStringBuilder Indent()    // Increases indent by 4 spaces
FormattedStringBuilder Unindent()  // Decreases indent by 4 spaces (throws if already at 0)
```

Each call to `Indent()` adds 4 spaces to the current indentation level. `Unindent()` removes 4 spaces.

### Appending Content

```csharp
FormattedStringBuilder Append(string value)       // Appends text at current cursor position (no indentation)
FormattedStringBuilder AppendLine()                // Appends newline + current indentation whitespace
FormattedStringBuilder AppendLine(string value)    // Appends text + newline + current indentation whitespace
FormattedStringBuilder AppendLine(char value)      // Appends char + newline + current indentation whitespace
```

**Key behavior:** Indentation is applied after the newline in `AppendLine()`, positioning the cursor for the *next* write. `Append()` writes at the current cursor position without adding indentation.

### Output

```csharp
string ToString()  // Returns the generated string
```

All `StringBuilder` methods (`AppendFormat`, `Insert`, `Replace`, `Clear`, etc.) are also available as delegating methods.

## Extension Methods

The `FormattedStringBuilderExtensions` class provides helpers for common source generator patterns:

```csharp
// Appends: #nullable enable
FormattedStringBuilder AppendNullableEnable()

// Appends: #pragma warning disable 0618, 1591, 8019
FormattedStringBuilder AppendCommonPragmaDisable()

// Appends: [global::System.CodeDom.Compiler.GeneratedCode("AssemblyName", "0.0.0.0")]
FormattedStringBuilder AppendGeneratedAttribute(string assemblyName, string assemblyVersion = "0.0.0.0")

// Appends: namespace MyNamespace;  (skips if namespace is null/whitespace)
FormattedStringBuilder AppendFileScopedNamespaceDeclaration(string? @namespace)

// Appends: typeof(typeName)
FormattedStringBuilder AppendTypeof(string typeName)

// Appends a raw string literal (""" ... """)
FormattedStringBuilder AppendRawString(string rawString)
```

## Example: Generating a Class

Here is how the library's `SourceTextSourceGenerator` uses `FormattedStringBuilder` to generate a class:

```csharp
var builder = new FormattedStringBuilder();

builder
    .AppendAutoGeneratedComment()
    .AppendNullableEnable()
    .AppendCommonPragmaDisable()
    .AppendLine()
    .AppendLine("using System.Text;")
    .AppendLine("using Microsoft.CodeAnalysis.Text;")
    .AppendLine()
    .AppendFileScopedNamespaceDeclaration("MyNamespace")
    .AppendLine()
    .AppendGeneratedAttribute("MyGenerator")
    .Append("internal static class FooSourceText")
    .AppendLine()
    .Append('{')
    .Indent()
    .AppendLine()
    .Append("public static readonly SourceText SourceText = SourceText.From(")
    .Indent()
    .AppendLine()
    .Append("\"content\",")
    .AppendLine()
    .Append("Encoding.UTF8")
    .Unindent()
    .AppendLine()
    .Append(");")
    .Unindent()
    .AppendLine()
    .Append('}')
    .AppendLine();
```

This produces:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

using System.Text;
using Microsoft.CodeAnalysis.Text;

namespace MyNamespace;

[global::Microsoft.CodeAnalysis.Embedded]
[global::System.CodeDom.Compiler.GeneratedCode("MyGenerator", "0.0.0.0")]
internal static class FooSourceText
{
    public static readonly SourceText SourceText = SourceText.From(
        "content",
        Encoding.UTF8
    );
}
```

## How Indentation Works

The pattern is:
1. `Append('{')` -- writes the brace at the current position
2. `Indent()` -- increases the indent level
3. `AppendLine()` -- writes newline + indentation (now 4 spaces deeper)
4. `Append("content")` -- writes at the indented position
5. `Unindent()` -- decreases the indent level
6. `AppendLine()` -- writes newline + indentation (back to original level)
7. `Append('}')` -- writes the closing brace at the original level

## Best Practices

- **Use `Append` for same-line content, `AppendLine` for newlines.** `Append` does not add indentation -- it writes inline. `AppendLine` moves to the next line with proper indentation.
- **Use extension methods for standard headers.** `AppendNullableEnable()`, `AppendCommonPragmaDisable()`, and `AppendGeneratedAttribute()` ensure consistent auto-generated file headers.
- **Always pair `Indent()` and `Unindent()`.** Every `Indent()` call should have a matching `Unindent()`. An unmatched `Unindent()` throws an `InvalidOperationException`.
