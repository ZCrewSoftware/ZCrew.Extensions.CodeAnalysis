# Emitting Attributes

When you mark an attribute type with `[Microsoft.CodeAnalysis.Embedded]`, the library generates a complete infrastructure for embedding the attribute's source text into consuming projects and parsing its usages at compile time.

## The Attribute Pipeline

For an attribute type, the library generates up to **6 files**:

| Generated File             | Purpose                                                                             |
|----------------------------|-------------------------------------------------------------------------------------|
| `{BaseName}SourceText`     | Static `SourceText` property containing the attribute's source code                 |
| `I{BaseName}DataBuilder`   | Interface with `With{X}` methods for each parameter, type parameter, and property   |
| `{BaseName}Constructor`    | Matches `AttributeData` to the correct constructor overload and invokes the builder |
| `{BaseName}Parameter`      | Static instances for each constructor parameter, tied to builder methods            |
| `{BaseName}TypeParameter`  | Static instances for each generic type parameter (only if generic)                  |
| `{BaseName}NamedParameter` | Static instances for each settable property (only if properties exist)              |

The `BaseName` is the attribute's type name with the `Attribute` suffix removed (e.g., `RegisterAttribute` becomes `Register`).

## Example: Defining an Embedded Attribute

Consider a `RegisterAttribute` that captures a service/implementation pair with a lifetime:

```csharp
using System;

namespace MyGenerator;

[Microsoft.CodeAnalysis.Embedded]
internal class RegisterAttribute<TService, TImplementation> : Attribute
{
    public RegisterAttribute(string lifetime) { }

    public bool Lazy { get; set; }
}
```

This produces the following generated files (in the `MyGenerator` namespace).

### Generated: `IRegisterDataBuilder`

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal interface IRegisterDataBuilder
{
    void WithLifetime(global::Microsoft.CodeAnalysis.TypedConstant constant);
    void WithService(global::Microsoft.CodeAnalysis.ITypeSymbol symbol);
    void WithImplementation(global::Microsoft.CodeAnalysis.ITypeSymbol symbol);
    void WithLazy(global::Microsoft.CodeAnalysis.TypedConstant constant);
}
```

Each method corresponds to one aspect of the attribute:
- **Constructor parameters** (`lifetime`) become `WithLifetime(TypedConstant)`
- **Type parameters** (`TService`, `TImplementation`) become `WithService(ITypeSymbol)`, `WithImplementation(ITypeSymbol)` (the `T` prefix is stripped)
- **Settable properties** (`Lazy`) become `WithLazy(TypedConstant)`

### Generated: `RegisterParameter`

Each constructor parameter gets a static instance that knows its type and how to call the corresponding builder method:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal class RegisterParameter
    : global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeParameter<IRegisterDataBuilder>
{
    public static readonly RegisterParameter Lifetime = new(
        "System.String",
        (builder, constant) => builder.WithLifetime(constant)
    );

    public RegisterParameter(
        string type,
        global::System.Action<IRegisterDataBuilder, global::Microsoft.CodeAnalysis.TypedConstant> valueAction
    ) : base(type, valueAction) { }
}
```

### Generated: `RegisterTypeParameter`

Each generic type parameter gets a static instance that delegates to the builder:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal sealed class RegisterTypeParameter
    : global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeTypeParameter<IRegisterDataBuilder>
{
    public static readonly RegisterTypeParameter Service = new(
        (builder, symbol) => builder.WithService(symbol)
    );
    public static readonly RegisterTypeParameter Implementation = new(
        (builder, symbol) => builder.WithImplementation(symbol)
    );

    public RegisterTypeParameter(
        global::System.Action<IRegisterDataBuilder, global::Microsoft.CodeAnalysis.ITypeSymbol> valueAction
    ) : base(valueAction) { }
}
```

### Generated: `RegisterNamedParameter`

Each settable property gets a static instance. The instances are also collected into a `NamedParameters` array used by the constructor:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal sealed class RegisterNamedParameter
    : global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeNamedParameter<IRegisterDataBuilder>
{
    private static readonly RegisterNamedParameter lazy = new(
        "Lazy",
        (builder, _, constant) => builder.WithLazy(constant)
    );

    internal static readonly global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeNamedParameter<IRegisterDataBuilder>[] NamedParameters =
    [
        lazy
    ];

    public RegisterNamedParameter(
        string name,
        global::System.Action<IRegisterDataBuilder, string, global::Microsoft.CodeAnalysis.TypedConstant> valueAction
    ) : base(name, valueAction) { }
}
```

### Generated: `RegisterConstructor`

The `RegisterConstructor` ties everything together. It references the static instances from the types above and matches incoming `AttributeData` to the correct constructor overload:

```csharp
// <auto-generated/>
#nullable enable
#pragma warning disable 0618, 1591, 8019

namespace MyGenerator;

[global::System.CodeDom.Compiler.GeneratedCode("ZCrew.Extensions.CodeAnalysis.CSharp", "0.0.0.0")]
internal sealed class RegisterConstructor
    : global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeConstructor<IRegisterDataBuilder>
{
    private static readonly RegisterConstructor constructor0 = new(
        [RegisterTypeParameter.Service, RegisterTypeParameter.Implementation],
        [
            RegisterParameter.Lifetime,
        ]
    );
    private static readonly RegisterConstructor[] Constructors =
    [
        constructor0
    ];

    public RegisterConstructor(
        global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeParameter<IRegisterDataBuilder>[] parameters
    ) : base(parameters, RegisterNamedParameter.NamedParameters) { }

    public RegisterConstructor(
        global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeTypeParameter<IRegisterDataBuilder>[] typeParameters,
        global::ZCrew.Extensions.CodeAnalysis.CSharp.AttributeParameter<IRegisterDataBuilder>[] parameters
    ) : base(typeParameters, parameters, RegisterNamedParameter.NamedParameters) { }

    public static void WithAttributeData(
        IRegisterDataBuilder builder,
        global::Microsoft.CodeAnalysis.AttributeData attributeData)
    {
        foreach (var constructor in Constructors)
        {
            if (constructor.IsCalledBy(attributeData))
            {
                constructor.Accept(builder, attributeData);
                return;
            }
        }
    }
}
```

## Implementing the Data Builder

You provide two types: a **data model** to hold the parsed values, and a **builder** that implements the generated interface.

### Data Model

```csharp
internal sealed class RegisterData
{
    public string? Lifetime { get; set; }
    public ITypeSymbol? Service { get; set; }
    public ITypeSymbol? Implementation { get; set; }
    public bool Lazy { get; set; }
}
```

### Builder

```csharp
using Microsoft.CodeAnalysis;

namespace MyGenerator;

internal sealed class RegisterDataBuilder : IRegisterDataBuilder
{
    private readonly RegisterData data = new();

    public RegisterData Build() => this.data;

    public void WithLifetime(TypedConstant constant)
    {
        this.data.Lifetime = constant.Value as string;
    }

    public void WithService(ITypeSymbol symbol)
    {
        this.data.Service = symbol;
    }

    public void WithImplementation(ITypeSymbol symbol)
    {
        this.data.Implementation = symbol;
    }

    public void WithLazy(TypedConstant constant)
    {
        this.data.Lazy = constant.Value is true;
    }
}
```

## Using the Generated Code in Your Generator

### Emitting the SourceText

In your `IIncrementalGenerator.Initialize`, emit the attribute definition so consuming projects can use it:

```csharp
context.RegisterPostInitializationOutput(static context =>
{
    context.AddSource("RegisterAttribute.g.cs", RegisterSourceText.SourceText);
});
```

### Parsing Attribute Usages

When processing symbols decorated with your attribute, use the generated constructor to parse the data:

```csharp
foreach (var attributeData in symbol.GetAttributes())
{
    var builder = new RegisterDataBuilder();
    RegisterConstructor.WithAttributeData(builder, attributeData);
    var data = builder.Build();

    // Use data.Lifetime, data.Service, data.Implementation, data.Lazy
}
```

## Best Practices

- **Use `internal` types.** Embedded attributes are injected into consuming projects, so keeping them `internal` avoids polluting the public API.
- **Keep attributes simple.** Stick to primitive types and `Type` for constructor parameters. Complex types cannot be expressed in attribute syntax.
- **Separate the data model from the builder.** Implement the `IDataBuilder` on a dedicated builder class that populates a plain data object. This keeps parsing logic isolated from your generator's business logic.
- **Handle multiple constructors.** If your attribute has overloaded constructors, the generated `Constructor` class handles matching automatically. Each overload gets its own static instance.
